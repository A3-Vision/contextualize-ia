Bem-vindo ao Contextualize IA

Este pacote é uma base para você usar com o agente da sua IDE (Cursor, VS Code, Antigravity…): um PRD em Markdown que você cola no chat para o assistente montar a estrutura do repositório de contexto — pastas, CONTEXT.md vivo (perguntas e respostas, não texto morto) e o que fizer sentido para o seu caso.

Explorador — pasta contexto/ Diagnóstico

O arquivo principal para colar no agente está em cliente/templates/PRD-BASE-AGENTE-IDE.md. Material aprofundado opcional em cliente/conteudo/; modelos em cliente/templates/. O PRD orienta o agente a criar o manual vivo em contexto/ (não altera cliente/). Depois de extrair o ZIP, abra home.html nesta pasta para voltar aqui.

Servidor local

Ver o pacote no navegador

Este front é HTML/CSS/JS estático; os caminhos partem da raiz do pacote (home.html → cliente/app/...). Abrir só com file:// às vezes quebra scripts, fontes ou o explorador (que usa fetch). Use um servidor HTTP na pasta raiz.

Opção 1 — Python 3

No PowerShell, na pasta raiz do pacote (onde está home.html):

Set-Location "caminho\completo\ate\a-pasta-do-pacote"
python -m http.server 8080

No macOS ou Linux, use cd no lugar de Set-Location. No navegador: http://localhost:8080/home.html. Para parar: Ctrl+C no terminal.

Opção 2 — Node (npx, sem instalar global)

Na mesma pasta raiz:

npx --yes serve -p 8080

O serve costuma listar a pasta; abra /home.html na URL indicada (ex.: http://localhost:8080/home.html).

Opção 3 — Sem terminal (só editor)

Extensão Live Server no VS Code ou Cursor: abra a pasta raiz do pacote como projeto e use “Go Live”. Confirme que a raiz servida é a do repositório para os links cliente/... funcionarem.

Resumo: suba o servidor na raiz onde está home.html e use /home.html como entrada. O explorador (cliente/app/explorador.html) continua precisando de HTTP e, se for o caso, do manifesto gerado com node cliente/scripts/gerar-estrutura-manifest.js, como no README do pacote.
